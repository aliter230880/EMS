# EMS Gaming Platform — Контекст проекта

## Суть идеи

**EMS Gaming Platform** — это система, которая объединяет браузерную/десктопную игру (раннер/платформер) с физической обратной связью через электростимуляцию мышц (EMS) или вибрацию. Человек играет в игру и одновременно получает физическую нагрузку: прыжок в игре → импульс на квадрицепсы, бег → ритмичные импульсы на икры.

Автор идеи: **Дима**

---

## Архитектура системы

```
[Игра (ПК/браузер)]
    │ HTTP/WebSocket
    ▼
[Python Bridge / API-сервер]
    │ BLE (Bluetooth Low Energy) / USB / HTTP API
    ▼
[Физическое устройство]
    • Телефон (vibrate API) — для PoC
    • Mi Band 8 — для BLE-теста
    • DIY EMS-модуль (ESP32 + TENS) — полноценный EMS
    • Чужие костюмы с переделанными разъёмами — промежуточный этап
    • Собственный EMS-костюм — финальная цель
```

**Ключевой принцип:** Игра изолирована от железа через посредника (Bridge), который:
- принимает HTTP/WebSocket команды от игры (`{"action": "jump", "intensity": 0.5}`)
- применяет safety-ограничения (кулдаун ≥ 1.5–2 сек, лимит интенсивности, никогда не стимулировать шею/грудь/голову)
- транслирует команды на физическое устройство

## Маппинг игровых событий → физический отклик

| Событие в игре     | Интенсивность | Мышцы             | Длительность |
|--------------------|---------------|-------------------|--------------|
| Бег                | Слабая (1-2)  | Икроножные        | Ритм 50–100 мс |
| Прыжок             | Средняя (2-3) | Квадрицепсы       | 200–500 мс   |
| Сбор монеты/бонус  | Слабая (1)    | Любые             | 20–50 мс     |
| Получение урона    | ❌ Нет        | —                 | —            |
| Приземление        | Короткая (2)  | Ноги              | 100 мс       |

## Технологический стек

### MVP (минимальный прототип)
- **Игра:** HTML5 + JavaScript (`navigator.vibrate()` на телефоне) или Unity WebGL
- **Bridge:** Python 3.x (Flask + Bleak для BLE) — `localhost:5000`
- **Устройство тест #0:** Telegram Bot API → push-уведомление → вибрация телефона
- **Устройство тест #1:** Xiaomi Mi Band 8 (обычная, не Active) — BLE через bleak

### Продакшн
- **Backend/Bridge:** Python (FastAPI) или Node.js
- **Протокол игра↔bridge:** WebSocket (задержка < 100–200 мс)
- **Игровой движок:** Unity (C#) / Godot
- **БД:** PostgreSQL (профили игроков, достижения)
- **Деплой:** Docker + VPS

## Этапы разработки

### Этап 0: Proof of Concept — ✅ ЗАВЕРШЁН (0 ₽, ~1 день)
- HTML5 игра-раннер (`game.html`) с `navigator.vibrate()`
- Открывается на Samsung Galaxy S10 в Chrome
- **Результат:** Вибрация работает, концепция подтверждена. Тап иногда не срабатывал → пофикшено (обработчик на весь экран)

### Этап 1: BLE-прототип (≈ 4 000 ₽, 3–5 дней)
- Купить Xiaomi Mi Band 8 (обычная, НЕ Active)
- Через nRF Connect найти MAC-адрес и UUID характеристики Write
- Получить Auth Token через Gadgetbridge
- Python-bridge (bleak): подключение + отправка команд вибрации
- Протестировать задержку (цель < 200 мс)
- **Примечание:** Mi Band 8 использует protobuf протокол — нужно правильно формировать сообщения

### Этап 2: DIY EMS-модуль (≈ 5 000–9 500 ₽, 1–2 недели)
**Вариант A (дешевле):**
- ESP32 (~500 ₽) + OEM TENS-модуль с CE-сертификатом (~3 000 ₽) + электроды (~500 ₽) + корпус (~500 ₽) = ~5 000 ₽

**Вариант B (безопаснее):**
- ESP32 (~500 ₽) + Beurer EM 80 (~8 000 ₽) = ~8 500 ₽

**КРИТИЧНО:** Схема обязательно должна иметь гальваническую развязку (оптроны + изолированный DC-DC преобразователь)

Прошивка ESP32: слушает WebSocket/HTTP от игры → генерирует безопасные импульсы (20–80 Гц, < 1 сек)

### Этап 3: Использование чужих костюмов с нашим модулем
- Готовые EMS-костюмы (Miha Bodytec, XBody) имеют **закрытый проприетарный протокол** — нельзя подключить напрямую
- **Решение:** Отрезаем их проприетарный разъём → припаиваем стандартные 2мм jack → подключаем наш ESP32-модуль
- Источники костюмов: Avito ("EMS костюм", 10 000–25 000 ₽), eBay ("XBody suit", $100–300), Alibaba оптом ($50–150/шт)

### Этап 4: Полноценная игра (2–4 недели)
- Unity-проект: раннер + платформер
- Свайп вверх = прыжок, свайп вниз = подкат
- Система прогрессии, достижения, таблица лидеров
- Интеграция через UnityWebRequest → Python Bridge

### Этап 5: Коммерциализация (3–6 месяцев)
- **B2C:** Игра + DIY-набор EMS = ~15 000 ₽ (конкуренты: 150 000+ ₽)
- **B2B:** Лицензирование технологии фитнес-клубам
- **SaaS:** SDK для Unity/Unreal/Godot ($99/лицензия) + Marketplace для EMS-игр

## Правила безопасности (ОБЯЗАТЕЛЬНО)

- ❌ Никогда не стимулировать шею, грудь (возле сердца), голову, виски
- ✅ Только крупные группы мышц: ноги, спина, пресс
- ✅ Начинать с минимальной интенсивности (уровень 1–2)
- ✅ Кулдаун между импульсами: ≥ 1.5–2 секунды
- ✅ Таймаут автоотключения: 0.5 сек после каждого импульса
- ✅ Использовать только сертифицированные TENS/EMS модули
- ✅ Гальваническая развязка в DIY-схеме (оптроны + изолированный DC-DC)
- ✅ Использовать "боль как механику" — никогда

## Ключевые примеры кода

### Python Bridge (Flask + Bleak)
```python
# pip install flask bleak
from flask import Flask, request, jsonify
import asyncio
from bleak import BleakClient
import time

app = Flask(__name__)
DEVICE_ADDRESS = "XX:XX:XX:XX:XX:XX"
VIBRATE_CHAR_UUID = "0000xxxx-0000-1000-8000-00805f9b34fb"
last_impulse_time = 0
COOLDOWN = 1.5

async def send_impulse_to_device(intensity_level):
    global last_impulse_time
    if time.time() - last_impulse_time < COOLDOWN:
        return  # Safety cooldown
    async with BleakClient(DEVICE_ADDRESS) as client:
        command = bytearray([0x01, intensity_level])
        await client.write_gatt_char(VIBRATE_CHAR_UUID, command)
        last_impulse_time = time.time()
        await asyncio.sleep(0.5)
        await client.write_gatt_char(VIBRATE_CHAR_UUID, bytearray([0x00, 0x00]))

@app.route('/api/impulse', methods=['POST'])
def trigger_impulse():
    data = request.json
    intensity = data.get('intensity', 1)
    asyncio.run(send_impulse_to_device(intensity))
    return jsonify({"status": "success"}), 200

if __name__ == '__main__':
    app.run(port=5000)
```

### Unity C# → Bridge
```csharp
using UnityEngine.Networking;

IEnumerator SendImpulse(int intensity, string action) {
    string json = $"{{\"intensity\":{intensity},\"action\":\"{action}\"}}";
    var request = UnityWebRequest.Post("http://localhost:5000/api/impulse", json, "application/json");
    yield return request.SendWebRequest();
}

void OnJump() { StartCoroutine(SendImpulse(2, "jump")); }
void OnRun()  { StartCoroutine(SendImpulse(1, "run"));  }
```

```python
# [Игра на ПК] → [Python] → [Telegram Bot API] → [Push на телефон] → Вибрация

    requests.post(url, json={"chat_id": chat_id, "text": message, "disable_notification": False})
```

## Риски

| Риск                       | Вероятность | Митигация                                               |
|----------------------------|-------------|---------------------------------------------------------|
| Электротравма              | Средняя     | Гальваническая развязка, лимиты в коде, тест на себе   |
| Юридические проблемы       | Высокая     | Дисклеймеры, CE/FDA-сертификация устройства             |
| Высокая задержка (> 500мс) | Средняя     | WebSocket вместо HTTP, локальный bridge                 |
| Дискомфорт от стимуляции   | Высокая     | Настройка интенсивности, разные режимы                  |
| Малая аудитория            | Высокая     | Начать с вибрации (масс-маркет), EMS — для энтузиастов  |
