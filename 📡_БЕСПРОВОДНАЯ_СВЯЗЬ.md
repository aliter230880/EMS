### Вариант A: **Полностью беспроводная система** (XBody PRO, Visionbody)

```
┌─────────────────────────────┐
│  ПЛАНШЕТ / ПРИЛОЖЕНИЕ       │
│  (управление тренером)      │
└──────────┬──────────────────┘
           │
           │ Bluetooth / Wi-Fi
           │ (ЗАКРЫТЫЙ ПРОТОКОЛ!)
           │
           ▼
┌─────────────────────────────────────────┐
│  POWERBOX (на теле, на поясе)           │
│                                         │
│  Внутри:                                │
│  • Батарея Li-Ion (2-15 часов)          │
│  • Bluetooth/Wi-Fi приёмник             │
│  • TENS-генератор (сердце системы)      │
│  • Микроконтроллер (обработка команд)   │
│                                         │
│  Получает команды:                      │
│  "Канал 1 (грудь): 50Hz, 200мкс, 30mA"  │
│  "Канал 2 (пресс): 80Hz, 150мкс, 25mA"  │
│  ...и т.д.                              │
└──────────┬──────────────────────────────┘
           │
           │ ПРОВОДА (внутри костюма)
           │
           ▼
┌─────────────────────────────┐
│  ЭЛЕКТРОДЫ (в ткани)        │
│  • Грудь                    │
│  • Пресс                    │
│  • Спина                    │
│  • Руки                     │
│  • Ноги                     │
└─────────────────────────────┘
```

#### Как это работает:

1. **Планшет** отправляет команду по Bluetooth/Wi-Fi
2. **Powerbox на теле** получает команду
3. **TENS-генератор** создаёт электрические импульсы
4. **Провода** (внутри костюма!) передают импульсы на электроды
5. **Электроды** стимулируют мышцы

#### ⚠️ ВАЖНО: Беспроводная только связь Планшет ↔ Powerbox!

**Powerbox → Электроды = ПРОВОДА** (внутри ткани костюма)

## 🔐 ПОЧЕМУ МЫ НЕ МОЖЕМ ПОДКЛЮЧИТЬСЯ?

### Причина 1: **ЗАКРЫТЫЙ ПРОТОКОЛ**

```
Планшет                    Powerbox
   │                          │
   │─────── Bluetooth ────────│
   │                          │
   │  Команда выглядит так:   │
   │  [0xA3, 0x4F, 0x12, ...] │ ← Зашифровано!
   │                          │
   └──────────────────────────┘
```

#### Что мы НЕ знаем:

- ❌ **Формат команд** (какие байты что означают)
- ❌ **Шифрование** (возможно, AES или проприетарное)
- ❌ **Аутентификация** (нужен ли ключ для подключения)
- ❌ **Протокол handshake** (как установить соединение)
- ❌ **Safety-логика** (как костюм проверяет безопасность команд)

#### Пример закрытого протокола:

```
// Мы пытаемся отправить:
send_bluetooth("Channel_1_On")  ❌ НЕ РАБОТАЕТ!

// А нужно отправить что-то типа:
send_raw_bytes([
  0xA3,  // Header
  0x4F,  // Device ID
  0x12,  // Command: Set Channel
  0x01,  // Channel 1
  0x32,  // Frequency: 50Hz
  0xC8,  // Pulse width: 200μs
  0x1E,  // Current: 30mA
  0x5A   // Checksum
])

НО: Мы не знаем эти байты! Производитель держит в секрете!
```

### Причина 2: **АУТЕНТИФИКАЦИЯ**

```
Планшет → Powerbox:
  "Привет, я планшет XBody с сертификатом ABC123"

Powerbox → Планшет:
  "Проверяю... Сертификат валиден. Подключаю."

───────────────────────────────────────────

Наш модуль → Powerbox:
  "Привет, я модуль!"

Powerbox → Наш модуль:
  "Кто ты? Где сертификат? Отключаюсь." ❌
```

#### Типы защиты:

1. **Bluetooth Pairing** (PIN-код)
2. **Certificate-based auth** (сертификат шифрования)
3. **Hardware ID check** (проверка серийного номера)
4. **Time-based tokens** (меняющиеся ключи)

---

### Причина 3: **ПОСТОЯННЫЕ ОБНОВЛЕНИЯ**

```
Месяц 1: Мы расшифровали протокол! 🎉

Месяц 2: XBody выпустил обновление прошивки
         → Протокол изменился
         → Наш код перестал работать ❌

Месяц 3: Мы снова расшифровали!

Месяц 4: Ещё одно обновление...
         → Бесконечная гонка ❌
```

Производители **специально** меняют протоколы, чтобы блокировать сторонние решения.

### Причина 4: **ЮРИДИЧЕСКИЕ РИСКИ**

#### Обратный инжиниринг закрытого протокола = нарушение закона:

```
❌ DMCA (Digital Millennium Copyright Act)
   "Обход технических средств защиты"

❌ Патентное право
   "Использование запатентованной технологии"

❌ Лицензионное соглашение
   "Нарушение ToS (Terms of Service)"
```

#### Последствия:

- Судебный иск от производителя
- Штрафы ($10,000 - $100,000+)
- Уголовная ответственность (в некоторых странах)

## 🤔 "НО ВЕДЬ ЭТО ПРОСТО BLUETOOTH!"

### Да, но...

```
Обычный Bluetooth:
  • Открытый стандарт (GATT, SPP, HID и т.д.)
  • Документированные профили
  • Любое устройство может подключиться

Bluetooth в EMS-костюмах:
  • ПОВЕРХ Bluetooth идёт проприетарный протокол
  • Формат команд ЗАКРЫТ
  • Нужна АУТЕНТИФИКАЦИЯ
  • Постоянно МЕНЯЕТСЯ
```

#### Аналогия:

```
Это как WhatsApp:
  • Работает через интернет (открытый стандарт)
  • НО: Протокол внутри ЗАКРЫТ
  • НЕ можешь написать свой клиент WhatsApp
  • НЕ можешь подключиться сторонним софтом
```

## ✅ ЧТО МЫ МОЖЕМ СДЕЛАТЬ?

### Вариант 1: **Man-in-the-Middle** (⚠️ Сложно и рискованно)

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│ Планшет  │────→│ Наш прокси│────→│ Powerbox │
└──────────┘     └──────────┘     └──────────┘
                      ↓
                 Перехватываем
                 и анализируем
                 команды
```

#### Проблемы:

- ❌ Нужны специальные инструменты (Ubertooth, HackRF)
- ❌ Нужны навыки реверс-инжиниринга
- ❌ Месяцы работы
- ❌ Юридические риски
- ❌ Производитель может изменить протокол

---
